=== diff ===
Three categories of divergence:

(1) Check 1/3 (PodSecurity) — Rust PASSes, bash FAILs.
    Root cause: bash twin's `kc apply -f - <<EOF` heredoc has a
    pre-existing stdin-handoff bug. The kc() wrapper invokes
    scripts/kubectl-k8s.sh which sets up a port-forward tunnel via
    `ssh -fNL …` BEFORE exec'ing kubectl. The setup phase reads
    from stdin (or otherwise consumes the heredoc) before kubectl
    sees it, so `kubectl apply -f -` gets an empty stdin and errors
    with `error: no objects passed to apply`. Bash's regex
    `printf '%s' "$ps_out" | grep -q 'violates PodSecurity'` then
    misses, and the check is marked FAIL.

    The Rust path SSHes the manifest directly through
    `hbird_ssh::Client::run_with_stdin` (no port-forward wrapper)
    so kubectl receives the heredoc verbatim and the apiserver
    correctly rejects it with the `violates PodSecurity` marker
    Rust's check 1 is looking for.

    Rust path is MORE correct than bash. Operator action:
    - Short-term: trust the Rust verifier when it disagrees.
    - Follow-up: fix bash twin's kc-via-kubectl-k8s.sh stdin
      handling — separate issue (verify-hardening + verify-app-deploy
      both rely on kc apply -f -).

(2) Rust emits one ERROR span event with the kubectl-stderr
    payload, between the `check 1/3` log line and the `PASS:` log
    line. Source: `#[tracing::instrument(... err(Debug))]` on
    `hbird_ssh::Client::run_inner`. When the remote kubectl exits
    non-zero (which IS the PASS condition here — PSA rejection
    surfaces as exit-1), hbird-ssh fires the instrument's `err`
    span event at ERROR level even though the verify-hardening
    callsite folds the NonZeroExit into a successful
    CpExecOutput { success: false, … } via
    `cp_kubectl_with_stdin_lenient`.

    Cosmetic only — interleaves with operator-grep-anchor lines on
    stderr. Tracked for a hbird-ssh tracing-level follow-up
    (consider demoting `err(Debug)` to `debug!(error = …)` so
    callers can decide their own ERROR-vs-debug policy).

(3) Bash has the `[verify-hardening] CP_IP not set, trying
    resolve_cp_ip hbird-cp1` line; Rust skips it because the live
    test passes `--cp-ip` explicitly. Both end up at the same
    CP_IP=192.168.122.212.

=== verdict: PARITY (Rust strictly more correct) ===
