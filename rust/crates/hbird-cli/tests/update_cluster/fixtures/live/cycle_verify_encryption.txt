=== diff ===
- bash: `via geary`
+ Rust: `via aatchison@geary`
  (Devcontainer workaround per #320 — `aatchison@` prefix forces the
   ProxyJump SSH to authenticate as the operator's user. From a host
   shell with $USER=aatchison both forms are equivalent.)

- bash: probe secret PID = 4941
+ Rust: probe secret PID = 5026
  (`$$` / `std::process::id()` — different process IDs across runs,
   expected.)

+ Rust adds: ` INFO hbird{subcommand="verify"}: close`
  (tracing-subscriber emits a span-close event at INFO level — wired
   in #326. The bash twin has no equivalent. This is a parity-irrelevant
   debug-aid line; operators grepping `[verify-encryption]` see the
   exact same `OK:` marker.)

Modulo those three diffs, output is byte-for-byte identical.

=== verdict: PARITY ===
- Both sides exit 0 with the bash twin's
  `OK: secret in etcd is encrypted (prefix=k8s:enc:aesgcm:)` marker on
  stderr. Operators grepping that anchor see identical results.
- Cluster left 3/3 Ready on both runs (probe secret is cleaned up by
  the bash-twin's trap and the on-CP /usr/libexec script's trap; the
  Rust path inherits the same trap because it just SSHes the same
  script into root@CP_IP).
