# Phase 3 cycle: `hbird get-kubeconfig` parity vs bash twin
#
# Captured 2026-05-27 against the live geary cluster (hbird-cp1).
# `get-kubeconfig` is the daily-use sibling of `export-argocd` — same
# fetch + rewrite path with operator-friendly defaults
# (`--output kubeconfig.yaml`, `--context-name=$CP_NAME`).
#
# bash side:
#   CONFIG=/tmp/cluster-live.conf bash scripts/export-argocd.sh \
#     --output /tmp/bash-kubeconfig.yaml --context-name=hbird-cp1 --force
# rust side:
#   hbird get-kubeconfig --config /tmp/cluster-live.conf \
#     --output /tmp/rust-kubeconfig.yaml --force
#
# Note: the bash twin is the same `export-argocd.sh` invoked with
# the operator-friendly defaults via `make get-kubeconfig`. The Rust
# `get-kubeconfig` subcommand delegates to the shared
# `ExportOptions::for_get_kubeconfig` + `export_kubeconfig` core, so
# the only behavioral difference from `cycle_export_argocd.txt` is
# the default context name (`hbird-cp1` here, `hummingbird-hbird-cp1`
# for export-argocd).
